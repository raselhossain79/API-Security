# Practice Methodology — README

This folder is different from the rest of the repo: it's not a vulnerability
reference, it's the workflow for actually USING everything else you've built.

## Files

| File | Covers |
|---|---|
| `00_How_To_Practice.md` | The practice loop, how much payload coverage is "enough" per priority tier, mechanism-first mindset, troubleshooting order |
| `01_Writeup_Template.md` | Copy-paste template for documenting every lab/session in real report format |

## Read This When

- Right now, once, before starting practice on any topic
- Again anytime you feel like you're "just collecting payloads" instead of
  actually understanding — that's the signal to re-read `00_How_To_Practice.md`

## The One-Sentence Summary

Read the theory once, break the easiest lab with minimal payloads while explaining
*why* each one works, solve it, write it up the same day using the template, then
move to the next lab — and don't force equal depth on low-priority topics.
